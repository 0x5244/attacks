# Glossary of Security Terms

## A

**API (Application Programming Interface)** - A set of protocols and tools for building software applications and enabling communication between systems.

**Authentication** - The process of verifying the identity of a user or system.

**Authorization** - The process of determining what actions an authenticated user is allowed to perform.

## B

**Brute Force Attack** - An attack method that tries all possible combinations to guess passwords or encryption keys.

**Buffer Overflow** - A vulnerability where more data is written to a buffer than it can hold, potentially allowing code execution.

## C

**CORS (Cross-Origin Resource Sharing)** - A security feature that controls how web pages from one domain can request resources from another domain.

**CSRF (Cross-Site Request Forgery)** - An attack that tricks a user into executing unwanted actions on a web application where they're authenticated.

**CI/CD (Continuous Integration/Continuous Deployment)** - Automated practices for integrating code changes and deploying applications.

## D

**DDoS (Distributed Denial of Service)** - An attack that overwhelms a system with traffic from multiple sources to make it unavailable.

**Deserialization** - The process of converting serialized data back into objects, which can be exploited if not properly secured.

## E

**Encryption** - The process of encoding data to prevent unauthorized access.

**Exploit** - A piece of code or technique that takes advantage of a vulnerability.

## I

**IAM (Identity and Access Management)** - Systems and policies for managing digital identities and controlling access to resources.

**IDOR (Insecure Direct Object Reference)** - A vulnerability where an application exposes references to internal objects without proper authorization checks.

**Injection** - A class of attacks where malicious data is sent to an interpreter as part of a command or query.

## K

**Kubernetes (K8s)** - An open-source container orchestration platform for automating deployment, scaling, and management of containerized applications.

## M

**MFA (Multi-Factor Authentication)** - Authentication requiring two or more independent credentials for verification.

**Microservices** - An architectural style where applications are composed of small, independent services.

## O

**OWASP (Open Web Application Security Project)** - A nonprofit foundation focused on improving software security.

## P

**Payload** - The malicious data or code delivered in an attack.

**Privilege Escalation** - Gaining higher access levels than originally granted.

## R

**RBAC (Role-Based Access Control)** - An access control method that assigns permissions based on user roles.

**RCE (Remote Code Execution)** - A vulnerability that allows an attacker to execute arbitrary code on a remote system.

## S

**SQLi (SQL Injection)** - An attack that inserts malicious SQL statements into application queries.

**SSRF (Server-Side Request Forgery)** - An attack that tricks a server into making requests to unintended locations.

**SSL/TLS (Secure Sockets Layer/Transport Layer Security)** - Cryptographic protocols for secure communication over networks.

## V

**Vulnerability** - A weakness in a system that can be exploited to compromise security.

**VPC (Virtual Private Cloud)** - An isolated virtual network within a cloud environment.

## X

**XSS (Cross-Site Scripting)** - An attack that injects malicious scripts into web pages viewed by other users.

**XXE (XML External Entity)** - An attack that exploits XML processors to access files or execute server-side requests.

## Z

**Zero-Day** - A vulnerability that is exploited before a patch or fix is available.

- **AAD** - Azure Active Directory
- **AWS** - Amazon Web Services
- **DoS** - Denial of Service
- **GCP** - Google Cloud Platform
- **HSTS** - HTTP Strict Transport Security

- **JWT** - JSON Web Token

- **PSP** - Pod Security Policy

- **XSSI** - Cross-Site Script Inclusion
