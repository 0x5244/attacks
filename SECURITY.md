# Security Policy

## Reporting a vulnerability

Please report suspected security vulnerabilities privately through GitHub's **Report a vulnerability** link on this repository's Security tab. Do not publish exploit details, credentials, tokens, personal data, or proof-of-concept secrets in a public issue.

Include the affected revision, impact, reproduction steps that do not disclose live data, and a safe contact method. We will acknowledge reports as soon as practical and coordinate remediation and disclosure.

## Secrets and sensitive data

Never commit credentials, access tokens, private keys, clinical records, or personal contact data. If a secret is exposed, revoke or rotate it immediately and report it privately so repository history and downstream copies can be assessed.

## Scope

This policy covers the default branch and published releases of this repository.
